Unidad 2


Estructura y funcionamiento de la unidad central de procesamiento

En Arquitectura de Computadoras, la Unidad Central de Procesamiento (CPU) es el componente fundamental que ejecuta las instrucciones de los programas.

2.1 Organización del procesador

La CPU se organiza en tres componentes principales:

  • Unidad de Control (UC): Coordina las operaciones de la CPU.
  • Unidad Aritmético-Lógica (ALU): Realiza operaciones matemáticas y lógicas.
  • Registros: Pequeñas memorias de alta velocidad para almacenamiento temporal.

La arquitectura clásica sigue el modelo de von Neumann, donde la CPU interactúa con la memoria principal para obtener instrucciones y datos.


2.2 Estructura de registros

Los registros son elementos clave en el diseño de la CPU:

2.2.1 Registros visibles para el usuario

  • Registros de propósito general: Pueden contener datos o direcciones (AX, BX, CX, DX en x86).
  • Registro acumulador: Almacena resultados de operaciones aritméticas.
  • Registros índice: Usados para direccionamiento indexado (SI, DI).
  • Registro puntero de pila (SP): Maneja la pila de llamadas a funciones.

2.2.2 Registros de control y de estados

  • Registro de instrucción (IR): Almacena la instrucción actual.
  • Contador de programa (PC): Indica la dirección de la próxima instrucción.
  • Registro de estado (FLAGS): Contiene bits que indican condiciones (cero, acarreo, signo, etc.).
  • Registro de dirección de memoria (MAR): Almacena direcciones para acceso a memoria.

2.2.3 Ejemplos de registros de CPU reales

En arquitectura x86:

  • EAX, EBX, ECX, EDX (32 bits)
  • ESI, EDI (índice)
  • ESP (puntero de pila), EBP (puntero base)
  • EIP (contador de instrucciones)
  • EFLAGS (registro de estado)

2.3 El ciclo de instrucción


2.3.1 Ciclo Fetch - Decode - Execute

El ciclo básico de ejecución de instrucciones consta de:

  1. Fetch (Búsqueda): Obtener la instrucción de memoria (PC → MAR → Memoria → IR).
  2. Decode (Decodificación): Interpretar la instrucción en la UC.
  3. Execute (Ejecución): La ALU realiza la operación especificada.
  4. Writeback (Escritura): Almacenar resultados en registros o memoria.

2.3.2 Segmentación de instrucciones

Técnica para mejorar el rendimiento dividiendo el ciclo en etapas que se ejecutan en paralelo:

  • Etapas típicas: Búsqueda, decodificación, ejecución, acceso a memoria, escritura.
  • Ventaja: Mayor throughput (más instrucciones por ciclo).
  • Desafíos: Riesgos de datos, control y estructurales.

2.3.3 Conjunto de instrucciones

Clasificación de arquitecturas según su conjunto de instrucciones (ISA):

  • CISC (Complex Instruction Set Computer): Muchas instrucciones complejas (x86).
  • RISC (Reduced Instruction Set Computer): Pocas instrucciones simples (ARM, MIPS).

2.3.4 Modos de direccionamiento

Formas de especificar operandos en las instrucciones:

  • Inmediato: El operando está en la instrucción (MOV AX, 5).
  • Directo: La instrucción contiene la dirección (MOV AX, [1234h]).
  • Indirecto: Un registro contiene la dirección (MOV AX, [BX]).
  • Indexado: Dirección = registro + desplazamiento (MOV AX, [SI+10h]).
  • Relativo a PC: Para saltos condicionales.

2.4 Casos de estudio de CPU reales

Ejemplos de arquitecturas modernas:

  • Intel x86: Evolución desde 8086 hasta Core i9, con extensiones como MMX, SSE, AVX.
  • ARM: Arquitectura RISC dominante en móviles, con conjunto de instrucciones Thumb.
  • MIPS: Arquitectura RISC clásica usada en enseñanza y sistemas embebidos.

Las CPUs modernas implementan características avanzadas como:

  • Ejecución fuera de orden (Out-of-Order execution)
  • Predicción de saltos (Branch prediction)
  • Múltiples núcleos (Multi-core)
  • Hyperthreading (ejecución simultánea de múltiples hilos)
